Т.к. отказались от Razor Pages нужен тот кто будет вызывать нужные вьюшки при вызове методов контроллеров. 
Для этого был написан движок обработки запросов на получение вьюх (страниц). Собственно это папка ViewEngine.

В appsettings.json есть настройки того, где движок будет искать страницы (.html)

В сборке, где лежат вьюшки надо создавать папку под каждый контроллер с тем же именем, что и контроллер.
	Внутри этой папки будет осуществляться поиск.
	В конечном итоге вьюшки будут искаться по следующему пути:
	<имяСборки_указатьВКонфигах>/<ИмяКорневойПапки_указатьВКонфигах>/<ИмяКонтроллера>/<ИмяМетода.html>

Как писать кастомные генераторы страниц (альтернативы Razor Page) можно посмотреть тут: https://metanit.com/sharp/aspnet5/7.10.php


Инструкция по запуску:
- добавить в Startup.cs->ConfigureServices следующий код:
	            services.Configure<MvcViewOptions>(options =>
            {
                options.ViewEngines.Clear(); // очищаем коллекцию обработчиков
                options.ViewEngines.Insert(0, new PageGeneratorEngine(Configuration)); // указываем наш обработчик
            });
- добавить в передаваемые в конструктор .cs конфигурации, которые нужны этому классу (поиск по файлу->Configuration)
	Например это можно сделать так: в appsettings.json добавить
		// Имя сборки где хранятся вьюшки (представления\фронтэнд)
		"ViewAssembly": "MakeHistory.View",
		// Папка в которой хранятся вьюшки (представления\фронтэнд)
		"ViewRootPath": "Views"
